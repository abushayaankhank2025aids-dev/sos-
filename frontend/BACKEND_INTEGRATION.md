# Backend Integration Guide

## Overview

This ARS (Alert Rescue System) dashboard is ready for backend integration. The frontend is fully functional with mock data and includes complete backend integration structure.

## Backend Data Format

The backend **MUST** provide SOS data in this exact format:

```json
{
  "latitude": 37.7749,
  "longitude": -122.4194,
  "message": "Emergency situation description",
  "battery": 85,
  "timestamp": "2026-05-03T10:30:00Z"
}
```

### Field Requirements:
- **latitude** (number): GPS latitude coordinate
- **longitude** (number): GPS longitude coordinate
- **message** (string): Full SOS message text
- **battery** (number): Battery percentage (0-100)
- **timestamp** (string): ISO 8601 timestamp

## Frontend Data Transformation

The frontend automatically derives these fields from backend data:

### Automatic Derivations:
1. **Priority** (`High` | `Medium` | `Low`)
   - Based on keywords in message + battery level
   - See `src/app/utils/sosUtils.ts:derivePriority()`

2. **Trigger Type** (`voice` | `manual`)
   - Detects "voice" keywords in message
   - See `src/app/utils/sosUtils.ts:deriveTriggerType()`

3. **AI Summary**
   - Generated from message content
   - See `src/app/utils/sosUtils.ts:generateAISummary()`

4. **Address** (optional)
   - Reverse geocoded from coordinates
   - Falls back to coordinate display if unavailable

5. **Status** (`Pending` | `Accepted` | `Reached` | `Resolved`)
   - Managed by frontend/backend state

6. **ID** (string)
   - Generated by frontend or provided by backend
   - Format: `SOS-{timestamp}` or custom from backend

## Backend API Endpoints Required

### REST API Endpoints:

#### 1. Get Active Alerts
```
GET /api/alerts/active
Response: BackendSOSData[]
```

#### 2. Get Alert History (Resolved)
```
GET /api/alerts/history
Response: BackendSOSData[]
```

#### 3. Update Alert Status
```
PUT /api/alerts/{alertId}/status
Body: {
  "status": "Pending" | "Accepted" | "Reached" | "Resolved",
  "rescuer": "Rescuer Name",
  "timestamp": "2026-05-03T10:30:00Z"
}
```

#### 4. Assign Alert to Rescuer
```
POST /api/alerts/{alertId}/assign
Body: {
  "rescuer": "Rescuer Name",
  "timestamp": "2026-05-03T10:30:00Z"
}
```

#### 5. Rescuer Authentication
```
POST /api/auth/login
Body: {
  "name": "John Doe",
  "phone": "+1234567890"
}
Response: { "success": true }
```

### WebSocket Endpoint:

#### Real-time SOS Feed
```
WS /ws/sos-feed

Server → Client messages: BackendSOSData (JSON)
```

## Environment Variables

Create a `.env` file in project root:

```env
REACT_APP_API_BASE_URL=http://localhost:8000/api
REACT_APP_WS_BASE_URL=ws://localhost:8000/ws
```

For production:
```env
REACT_APP_API_BASE_URL=https://your-backend.com/api
REACT_APP_WS_BASE_URL=wss://your-backend.com/ws
```

## Integration Steps

### Step 1: Backend Setup
1. Ensure backend provides the exact data format above
2. Implement all required API endpoints
3. Set up WebSocket server for real-time alerts

### Step 2: Environment Configuration
1. Create `.env` file with backend URLs
2. Verify URLs are correct

### Step 3: Enable Backend Calls
In `src/app/components/Dashboard.tsx`:
1. Uncomment the import:
   ```ts
   import { SOSWebSocket, fetchActiveAlerts, fetchAlertHistory, updateAlertStatus } from '../services/api';
   ```
2. Uncomment WebSocket initialization in `useEffect`
3. Uncomment data fetching in auto-refresh `useEffect`
4. Uncomment backend call in `handleUpdateStatus`

### Step 4: Test Connection
1. Start backend server
2. Start frontend: `pnpm dev`
3. Check browser console for connection status
4. Verify system status indicator shows "Live System Active" (green)

## Priority Derivation Logic

### High Priority:
- Keywords: medical, emergency, fire, flood, trapped, evacuation
- Battery < 20%

### Medium Priority:
- Keywords: stranded, supplies, elderly, blocked
- Battery 20-50%

### Low Priority:
- Infrastructure reports
- Battery > 50%

See full logic in `src/app/utils/sosUtils.ts`

## Troubleshooting

### WebSocket not connecting
1. Check `REACT_APP_WS_BASE_URL` in `.env`
2. Verify backend WebSocket server is running
3. Check browser console for connection errors
4. Ensure CORS is configured on backend

### Data not updating
1. Check `REACT_APP_API_BASE_URL` in `.env`
2. Verify backend REST API endpoints
3. Check network tab in browser DevTools
4. Ensure backend returns correct data format

### Status updates not persisting
1. Verify `PUT /api/alerts/{id}/status` endpoint
2. Check request payload format
3. Ensure backend saves status changes
